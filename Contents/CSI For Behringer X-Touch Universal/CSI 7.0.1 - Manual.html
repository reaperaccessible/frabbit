3. Configuration des surfaces de contrôle

Désactiver les ports MIDI

Avant de configurer CSI, vous devez désactiver les ports MIDI utilisés par votre surface de contrôle dans REAPER :

  1. Ouvrez les préférences REAPER avec Ctrl+P.
  2. Allez dans l'onglet MIDI Devices.
  3. Repérez les ports MIDI correspondant à votre surface (X-Touch Universal ou FCB1010).
  4. Désactivez-les en décochant les cases Enable input et Enable output pour chaque port.
  5. Cliquez sur Apply.

Désinstaller CSI 2.0 (migration vers 7.0.1)

Si vous utilisez CSI 2.0 et souhaitez passer à la version 7.0.1, vous devez d'abord désinstaller l'ancienne version :

  1. Ouvrez REAPER.
  2. Ouvrez les préférences avec Ctrl+P et allez dans Control/OSC/Web.
  3. Appuyez sur Tab jusqu'à la liste qui contient votre configuration de surface de contrôle et appuyez sur flèche bas pour la sélectionner.
  4. Appuyez sur Tab jusqu'au bouton Remove et appuyez sur Espace.
  5. Appuyez sur Shift+Tab jusqu'au bouton OK et appuyez sur Entrée.
  6. Fermez REAPER.
  7. Sélectionnez et copiez la ligne suivante : %appdata%
  8. Appuyez sur Windows+R, appuyez sur Ctrl+V pour coller et appuyez sur Entrée.
  9. Appuyez sur R pour sélectionner le dossier REAPER et appuyez sur Entrée.
  10. Appuyez sur U pour sélectionner le dossier UserPlugins et appuyez sur Entrée.
  11. Localisez le fichier reaper_csurf_integrator.dll et supprimez-le.
  12. Appuyez sur Retour arrière pour remonter d'un niveau, puis appuyez sur C pour sélectionner le dossier CSI.
  13. Déplacez ce dossier en lieu sûr si vous avez des fichiers .zon personnalisés que vous souhaitez réutiliser avec CSI 7.0.1.

Ajouter CSI comme surface de contrôle

  1. Ouvrez les préférences REAPER avec Ctrl+P.
  2. Allez dans Control/OSC/Web.
  3. Cliquez sur Add.
  4. Dans la liste déroulante, sélectionnez Control Surface Integrator.
  5. L'écran de préférences CSI s'affiche.

Configurer la surface dans l'écran de préférences CSI

  1. Appuyez sur Tab pour atteindre la liste des surfaces de contrôle disponibles. Par défaut, cette liste est vide. Vous devez ajouter une surface en utilisant le bouton Add MIDI, accessible avec Tab.
    1. Dans la fenêtre Add MIDI, le premier champ d'édition est le nom de la surface que vous souhaitez ajouter. Écrivez le nom (par exemple « X-Touch Universal » ou « FCB1010 »).
    2. Appuyez sur Tab et sélectionnez le port MIDI IN correspondant à votre surface.
    3. Appuyez sur Tab et sélectionnez le port MIDI OUT correspondant à votre surface.
    4. Appuyez sur Tab jusqu'à Number of Channels. Ce champ correspond au nombre de faders disponibles sur votre surface de contrôle : indiquez 8 pour la X-Touch Universal (8 faders) ou 2 pour la Behringer FCB1010 (2 pédales d'expression).
    5. Appuyez sur Tab jusqu'au bouton OK et appuyez sur Entrée.
  2. Retournez dans la liste des surfaces de contrôle et sélectionnez la surface que vous venez d'ajouter.
  3. Appuyez sur Tab pour atteindre la liste des pages de configuration CSI. Par défaut, il y en a une seule nommée Home. Vous pouvez utiliser autant de surfaces que vous le souhaitez simultanément au sein d'une même page.
  4. Appuyez sur Tab pour atteindre la liste des zones disponibles. Une zone est un ensemble de fichiers .zon qui définissent le comportement de votre surface de contrôle : quels boutons déclenchent quelles actions, quels faders contrôlent quels paramètres, etc. (voir la section 4 pour plus de détails). Cette liste est vide par défaut. Ajoutez une zone avec le bouton Add Zone et sélectionnez le dossier de zones correspondant à votre surface.
  5. Une fois la zone ajoutée, retournez dans la liste des zones et sélectionnez celle qui correspond à la surface sélectionnée dans la liste des surfaces.
  6. Appuyez sur Tab jusqu'au bouton OK et appuyez sur Entrée.
  7. Appuyez sur Tab jusqu'au bouton OK et appuyez sur Entrée.

Si tout s'est déroulé correctement, votre surface de contrôle devrait être fonctionnelle.

Activer les retours de surface dans OSARA

Important : vous devez activer les retours de surface de contrôle dans les paramètres OSARA pour que la vocalisation fonctionne correctement. Dans les préférences OSARA, cochez la case Report changes made via control surfaces.

Pages

Les Pages sont des configurations commutables dans CSI. Chaque Page peut avoir ses propres assignations de surface, zones et comportements. Vous pouvez basculer entre les Pages pendant votre session de travail.

Fichier csi.ini

Le fichier csi.ini est généré automatiquement par l'écran de préférences. Il stocke toute la configuration de vos surfaces. Il est situé dans le dossier CSI de REAPER.

3. Setting Up Control Surfaces

Disable MIDI Ports

Before configuring CSI, you must disable the MIDI ports used by your control surface in REAPER:

  1. Open REAPER Preferences with Ctrl+P.
  2. Go to the MIDI Devices tab.
  3. Find the MIDI ports for your surface (X-Touch Universal or FCB1010).
  4. Disable them by unchecking Enable input and Enable output for each port.
  5. Click Apply.

Uninstall CSI 2.0 (Migration to 7.0.1)

If you are using CSI 2.0 and want to upgrade to version 7.0.1, you must first uninstall the old version:

  1. Open REAPER.
  2. Open Preferences with Ctrl+P and go to Control/OSC/Web.
  3. Press Tab until the list containing your control surface configuration and press down arrow to select it.
  4. Press Tab until the Remove button and press Space.
  5. Press Shift+Tab until the OK button and press Enter.
  6. Close REAPER.
  7. Select and copy the following line: %appdata%
  8. Press Windows+R, press Ctrl+V to paste and press Enter.
  9. Press R to select the REAPER folder and press Enter.
  10. Press U to select the UserPlugins folder and press Enter.
  11. Locate the file reaper_csurf_integrator.dll and delete it.
  12. Press Backspace to go up one level, then press C to select the CSI folder.
  13. Move this folder to a safe location if you have custom .zon files that you wish to reuse with CSI 7.0.1.

Add CSI as a Control Surface

  1. Open REAPER Preferences with Ctrl+P.
  2. Go to Control/OSC/Web.
  3. Click Add.
  4. From the dropdown list, select Control Surface Integrator.
  5. The CSI preferences screen opens.

Configure the Surface in the CSI Preferences Screen

  1. Press Tab to reach the list of available control surfaces. By default, this list is empty. You must add a surface using the Add MIDI button, accessible with Tab.
    1. In the Add MIDI window, the first edit field is the name of the surface you want to add. Type the name (e.g. "X-Touch Universal" or "FCB1010").
    2. Press Tab and select the MIDI IN port for your surface.
    3. Press Tab and select the MIDI OUT port for your surface.
    4. Press Tab to Number of Channels. This field is the number of faders available on your control surface: enter 8 for the X-Touch Universal (8 faders) or 2 for the Behringer FCB1010 (2 expression pedals).
    5. Press Tab to the OK button and press Enter.
  2. Return to the control surface list and select the surface you just added.
  3. Press Tab to reach the CSI configuration pages list. By default, there is one page named Home. You can use as many surfaces as you want simultaneously within a single page.
  4. Press Tab to reach the list of available zones. A zone is a set of .zon files that define your control surface's behavior: which buttons trigger which actions, which faders control which parameters, etc. (see section 4 for details). This list is empty by default. Add a zone using the Add Zone button and select the zone folder corresponding to your surface.
  5. Once the zone is added, return to the zone list and select the one that corresponds to the surface selected in the surface list.
  6. Press Tab to the OK button and press Enter.
  7. Press Tab to the OK button and press Enter.

If everything went correctly, your control surface should be functional.

Enable Surface Feedback in OSARA

Important: you must enable control surface feedback in OSARA settings for vocalization to work correctly. In OSARA Preferences, check the Report changes made via control surfaces option.

Pages

Pages are switchable configurations in CSI. Each Page can have its own surface assignments, zones, and behaviors. You can switch between Pages during your work session.

csi.ini File

The csi.ini file is automatically generated by the preferences screen. It stores all your surface configuration and is located in REAPER's CSI folder.

4. Concepts fondamentaux

4.1 Zones

Les Zones sont le cœur de CSI. Ce sont des fichiers texte (extension .zon) qui associent les widgets de votre surface (boutons, faders, encodeurs) à des actions REAPER.

Types de zones

Home
Zone principale chargée au démarrage. Contient généralement les IncludedZones et les GoZones.
Track
Contrôle les pistes affichées sur les faders de la surface.
SelectedTrack
Contrôle la piste sélectionnée.
MasterTrack
Contrôle la piste Master.
FX Zones
Contrôle les paramètres des plugins.

Structure d'un fichier .zon

Zone "Home"
  IncludedZones
    "Track"
    "Buttons"
    "Transport"
  IncludedZonesEnd
ZoneEnd

GoZone et SubZone

GoZone permet de basculer vers une autre zone. GoSubZone superpose temporairement une zone par-dessus la zone actuelle, et LeaveSubZone la referme.

4.2 Zones FX

Les Zones FX associent les widgets de la surface aux paramètres d'un plugin spécifique. Chaque plugin nécessite son propre fichier .zon.

Zone "VST3: MonPlugin"
  FXParam 0 "Volume"    Fader1
  FXParam 1 "Pan"       Rotary1
  FXParam 2 "Bypass"    Button1 [ 0.0 1.0 ]
ZoneEnd

Les valeurs entre crochets [ 0.0 1.0 ] définissent un toggle (bascule) entre deux états.

Méthodes d'activation des zones FX

FocusedFX
Active automatiquement la zone quand la fenêtre du plugin a le focus.
TrackFXMenu / SelectedTrackFXMenu
Affiche un menu pour sélectionner le plugin à contrôler.
SelectedTrackFX
Active les zones FX de la piste sélectionnée.

4.3 Fichiers Surface.txt

Le fichier surface.txt définit les capacités de votre surface de contrôle : les widgets disponibles, les messages MIDI qu'ils envoient et reçoivent.

4.4 Modificateurs

Les modificateurs permettent d'assigner plusieurs actions à un même bouton :

Shift, Option, Control, Alt
Modificateurs globaux disponibles sur toute la surface.
Flip, Marker, Nudge, Scrub, Zoom
Modificateurs locaux spécifiques.
Touch
Activé quand un fader est touché.
InvertFB
Inverse le retour visuel (feedback).

4. Core Concepts

4.1 Zones

Zones are the core of CSI. They are text files (.zon extension) that map your surface's widgets (buttons, faders, encoders) to REAPER actions.

Zone Types

Home
Main zone loaded at startup. Usually contains IncludedZones and GoZones.
Track
Controls the tracks displayed on the surface's faders.
SelectedTrack
Controls the selected track.
MasterTrack
Controls the Master track.
FX Zones
Controls plugin parameters.

.zon File Structure

Zone "Home"
  IncludedZones
    "Track"
    "Buttons"
    "Transport"
  IncludedZonesEnd
ZoneEnd

GoZone and SubZone

GoZone switches to another zone. GoSubZone temporarily overlays a zone on top of the current one, and LeaveSubZone closes it.

4.2 FX Zones

FX Zones map surface widgets to specific plugin parameters. Each plugin requires its own .zon file.

Zone "VST3: MyPlugin"
  FXParam 0 "Volume"    Fader1
  FXParam 1 "Pan"       Rotary1
  FXParam 2 "Bypass"    Button1 [ 0.0 1.0 ]
ZoneEnd

Values in brackets [ 0.0 1.0 ] define a toggle between two states.

FX Zone Activation Methods

FocusedFX
Automatically activates the zone when the plugin window has focus.
TrackFXMenu / SelectedTrackFXMenu
Shows a menu to select which plugin to control.
SelectedTrackFX
Activates FX zones for the selected track.

4.3 Surface.txt Files

The surface.txt file defines your control surface capabilities: available widgets, the MIDI messages they send and receive.

4.4 Modifiers

Modifiers allow assigning multiple actions to the same button:

Shift, Option, Control, Alt
Global modifiers available across the entire surface.
Flip, Marker, Nudge, Scrub, Zoom
Specific local modifiers.
Touch
Activated when a fader is touched.
InvertFB
Inverts visual feedback.

5. Compatibilité OSARA et modes d'automatisation

Problème connu : Il existe un problème de comportement entre les surfaces de contrôle, certains modes d'automatisation et OSARA. L'équipe n'a pas encore déterminé si le problème vient d'OSARA, de REAPER ou de l'API des surfaces de contrôle. La seule certitude est que le problème ne vient pas des surfaces elles-mêmes, car tout le monde a le même problème, même si la surface n'est pas branchée.

Symptômes

Le problème est sévère : il annonce la valeur du premier point de n'importe quel type d'enveloppe, même lorsque vous :

  • Appuyez sur F4 pour ouvrir la liste des actions
  • Ouvrez la fenêtre FX
  • Naviguez entre les onglets des préférences

Si vous activez une enveloppe de volume, OSARA dira « 0.00 » ; une enveloppe de Pan, OSARA dira « Center », etc.

Modes qui ne causent pas le problème

Seuls 2 modes d'automatisation ne causent pas ce problème :

  • Trim/Read (recommandé)
  • Write

Solution recommandée

Restez toujours en mode Trim/Read sauf lorsque vous écrivez de l'automatisation.

Procédure pour rester en Trim/Read en permanence :

  1. Dans les Préférences, sélectionnez l'onglet Track/Send Defaults.
    • Appuyez sur Tab jusqu'à trouver « HardWare Output: Default automation mode: »
    • C'est une liste déroulante : sélectionnez Trim/Read.
  2. Dans les Préférences, sélectionnez l'onglet Automation.
    • Appuyez sur Tab jusqu'à trouver « After recording automation in write mode, on repeat/seek/stop »
    • Sélectionnez Switch to Trim/Read.

5. OSARA and Automation Mode Compatibility

Known issue: There is a behavior problem between control surfaces, certain automation modes, and OSARA. The team has not yet determined if the problem comes from OSARA, REAPER, or the control surface API. The only certainty is that it does not come from the surfaces themselves, as everyone has the same problem even when the surface is not plugged in.

Symptoms

The problem is severe: it announces the first point value of any envelope type, even when you:

  • Press F4 to open the Action list
  • Open the FX window
  • Navigate through Preferences tabs

If you activate a volume envelope, OSARA will say "0.00"; a Pan envelope, OSARA will say "Center", etc.

Modes That Do Not Cause the Problem

Only 2 automation modes do not cause this problem:

  • Trim/Read (recommended)
  • Write

Recommended Workaround

Always stay in Trim/Read automation mode except when writing automation.

Procedure to stay in Trim/Read permanently:

  1. In Preferences, select the Track/Send Defaults tab.
    • Press Tab until you find "HardWare Output: Default automation mode:"
    • It is a drop-down list: select Trim/Read.
  2. In Preferences, select the Automation tab.
    • Press Tab until you find "After recording automation in write mode, on repeat/seek/stop"
    • Select Switch to Trim/Read.

6. Fonctionnalités

6.1 Vue Globale (Global View)

Plusieurs actions ont été ajoutées dans la Vue Globale. Activez le mode d'aide pour découvrir toutes les nouvelles fonctionnalités.

Une aide supplémentaire a été ajoutée quand une action est assignée à un bouton en combinaison avec un modificateur (Shift, Option, Control ou Alt). NVDA vous informera qu'une combinaison est disponible.

6.2 Page d'édition

Vous pouvez maintenant utiliser le X-Touch pour éditer les items de vos pistes.

Comment accéder à la page d'édition :

  1. Sur la rangée en dessous de l'écran, appuyez sur le premier bouton à gauche (MidiTracks) pour ouvrir la page d'édition.
  2. Appuyez sur NameValue (bouton à gauche de l'écran) pour activer l'aide d'édition.
  3. Appuyez à nouveau sur MidiTracks pour quitter la vue d'édition, ou sur GlobalView pour retourner à la Vue Globale.

Note : Vous devez installer les scripts ReaperAccessible (pas besoin d'assigner des raccourcis clavier, il suffit de les charger dans REAPER). Merci à Armando pour sa contribution à la page d'édition.

6.3 Verrouillage des modificateurs

L'état de verrouillage des modificateurs est maintenant vocalisé :

  • Appui rapide sur un modificateur (Shift, Option, Control, Alt ou Flip) : il se verrouille (reste enfoncé). NVDA annonce « Lock ».
  • Appui rapide à nouveau : il se déverrouille. NVDA annonce « Unlock ».
  • Appui prolongé : « Lock » n'est pas annoncé, mais dès que vous relâchez, « Unlock » sera annoncé.

Cela vous permet d'utiliser des actions combinées avec un modificateur sans avoir à le maintenir enfoncé.

6.4 Enveloppes

Plusieurs actions liées aux enveloppes ont été ajoutées sur le X-Touch.

Pour y accéder :

  1. Appuyez sur NameValue pour activer le mode d'aide.
  2. Appuyez rapidement sur Control pour le verrouiller.
  3. Toutes les actions liées aux enveloppes sont en combinaison avec le modificateur Control.
  4. Appuyez à nouveau rapidement sur Control pour le déverrouiller et revenir au mode normal.

Actions disponibles :

Actions d'enveloppes sur le X-Touch
Combinaison Action
Control+JogWheel Gauche/Droite Diminuer/Augmenter la valeur du point d'enveloppe de 1
Control+Zoom Basculer le bypass de l'enveloppe
Control+Read Afficher/Masquer toutes les enveloppes de la piste sélectionnée

Générateur de fichiers .zon (Version 6.0)

Le générateur est un outil en ligne de commande Python bilingue (FR/EN) qui transforme les templates remplis en fichiers .zon prêts à l'emploi pour CSI. Il est optimisé pour les utilisateurs de lecteurs d'écran.

Fonctionnalités principales

  • Génération automatique des fichiers .zon avec navigation entre pages (Up/Down) et sections (Left/Right)
  • Génération optionnelle de zones Help qui vocalisent chaque contrôle via NVDA
  • Vocalisation du nom de la page, de la section et du plugin pendant la navigation
  • Noms de pages et sections personnalisables via les commentaires // page: "nom" et // section: "nom"
  • Validation automatique des fichiers .zon générés
  • Détection automatique de la langue (FR/EN)
  • Messages de progression adaptés aux lecteurs d'écran

Utilisation

python cli.py [options]

Options disponibles

--with-help
Générer les zones Help pour chaque page et section.
--dry-run
Afficher ce qui serait généré sans écrire de fichiers.
--batch
Mode non-interactif, sans aucune demande de confirmation.
--zone-name "nom"
Nom du plugin (évite la demande interactive).
--validate-only "dossier"
Valider un dossier de fichiers .zon existant sans rien générer.
--speak-test
Afficher toutes les actions Speak qui seraient générées.
-l fr ou -l en
Forcer la langue d'affichage.
-o "dossier"
Dossier de sortie.

  2. Move a parameter in the plugin.
  3. Click Link to map them.

Manual Creation

You can also create .zon files manually by editing text files. The basic structure is:

Zone "VST3: PluginName (Manufacturer)"
  FXParam 0 "ParamName"  Fader1
  FXParam 1 "ParamName"  Rotary1
  FXParam 2 "ParamName"  Button1 [ 0.0 1.0 ]
ZoneEnd

Stepped Parameters

For parameters with discrete values (dropdown lists, presets):

FXParam 5 "Type" Rotary3 [ 0.0 0.25 0.5 0.75 1.0 ]

Finding FX Parameter Numbers

Use the Write Params to ZoneRawFXFiles scripts to automatically generate a file with all plugin parameters and their numbers.

.zon File Generator (Version 6.0)

The generator is a bilingual (FR/EN) Python command-line tool that transforms filled-in templates into ready-to-use .zon files for CSI. It is optimized for screen reader users.

Key Features

  • Automatic .zon file generation with page navigation (Up/Down) and section navigation (Left/Right)
  • Optional Help zone generation that vocalizes each control via NVDA
  • Page name, section name, and plugin name vocalization during navigation
  • Customizable page and section names via // page: "name" and // section: "name" comments
  • Automatic validation of generated .zon files
  • Automatic language detection (FR/EN)
  • Screen reader friendly progress messages

Usage

python cli.py [options]

Available Options

--with-help
Generate Help zones for each page and section.
--dry-run
Show what would be generated without writing files.
--batch
Non-interactive mode, no confirmation prompts.
--zone-name "name"
Plugin name (skips the interactive prompt).
--validate-only "folder"
Validate an existing .zon folder without generating anything.
--speak-test

Complete Creation Process

  1. Copy a 16X16 template for your control surface.
  2. Identify your plugin's parameters with the Write Params to ZoneRawFXFiles script.
  3. Fill in the template sections by replacing NoAction with FXParam entries.
  4. Customize page and section names with // page: "name" comments.
  5. Delete unused pages and sections.
  6. Run the generator: python cli.py --with-help
  7. Copy the output folder to your CSI Zones directory.

14. Dépannage

CSI n'apparaît pas dans les préférences

  • Vérifiez que le fichier .dll est bien dans le dossier UserPlugins.
  • Assurez-vous d'utiliser la version 64 bits de REAPER.
  • Installez le Microsoft Visual C++ Redistributable x64.

La surface n'est pas reconnue

  • Vérifiez que les ports MIDI de la surface sont désactivés dans REAPER (onglet MIDI Devices des préférences).
  • Vérifiez que la surface est en mode MCU (Mackie Control).
  • Redémarrez REAPER après avoir modifié la configuration.

Les modifications de zones ne prennent pas effet

  • Après avoir modifié un fichier .zon, utilisez l'action Refresh All Surfaces dans REAPER.

Signaler un bug

Lorsque vous signalez un bug, incluez :

  • Votre système d'exploitation et version
  • Version de REAPER
  • Version de CSI
  • Étapes pour reproduire le problème

14. Troubleshooting

CSI Doesn't Appear in Preferences

  • Verify the .dll file is in the UserPlugins folder.
  • Make sure you're using the 64-bit version of REAPER.
  • Install Microsoft Visual C++ Redistributable x64.

Surface Not Recognized

  • Check that MIDI ports for the surface are disabled in REAPER (MIDI Devices tab in Preferences).
  • Verify the surface is in MCU (Mackie Control) mode.
  • Restart REAPER after changing configuration.

Zone Changes Not Taking Effect

  • After modifying a .zon file, use the Refresh All Surfaces action in REAPER.

Reporting a Bug

When reporting a bug, include:

  • Your operating system and version
  • REAPER version
  • CSI version
  • Steps to reproduce the issue
